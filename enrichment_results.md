## Implementation Notes

### How Company Inference Works

1. Extract domain from email (e.g., `a.martin@calamoycran.com` → `calamoycran.com`)
2. Skip generic domains (gmail, outlook, etc.)
3. Infer company name from domain (e.g., `calamoycran` → `Cálamo y Cran`)
4. Confirm via web search: `"{domain}" company linkedin`
5. Use inferred company in precision query: `site:linkedin.com/in "{name}" "{company}"`

### Why Precision Queries Work

- **Name-only search**: 20-50 results, high ambiguity (common names like "Juan García")
- **Name + company search**: 1-5 results, precise match almost guaranteed
- **Email domain inference**: Automatically narrows to ~1 likely employer
- **Multiple platforms**: LinkedIn, GitHub, personal sites all indexed by Serper

### Confidence Score Breakdown

- **0.9-1.0**: Found complete profile with job title, company, education, location
- **0.7-0.9**: Found profile with most fields; some inferred
- **0.4-0.7**: Found partial profile; requires manual verification
- **<0.4**: No profile found or name too ambiguous
- **Needs Review**: Flag set if confidence <0.5 or name has 3+ LinkedIn matches
